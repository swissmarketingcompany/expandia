# Quick Cornerstone Content Checklist

Use this checklist when creating new comprehensive blog posts based on the Business Development guide template.

## âœ… Pre-Writing Checklist

- [ ] Keyword research completed (primary + 5-10 related keywords)
- [ ] Competitor content analyzed (what exists, what's missing)
- [ ] 8-10 main sections outlined
- [ ] Target word count: 5,000+ words
- [ ] Stock photos sourced (6-8 images from Unsplash/Pexels)
- [ ] Internal links planned (8-12 service/content pages)
- [ ] External links identified (3-5 authority sources)

## âœ… Structure Checklist

- [ ] **Header**: Badge, H1 title, subtitle, meta info (date, reading time, word count)
- [ ] **Quick Navigation**: Table of contents with anchor links
- [ ] **Introduction**: Alert box + lead paragraph + 2-3 context paragraphs + stats dashboard
- [ ] **Hero Image**: High-quality photo after introduction
- [ ] **7-10 Main Sections**: Each with H2, image, 500-800 words, examples
- [ ] **Visual Elements**: 5-7 section images, 2-3 diagrams/charts, icons throughout
- [ ] **Interactive Elements**: Self-assessment, collapsible details, comparison tables
- [ ] **Mid-Article CTA**: ~40% through content
- [ ] **Assessment Form**: ~70% through content  
- [ ] **FAQ Section**: 8-10 questions with detailed answers
- [ ] **Related Resources**: Books + internal service links
- [ ] **Conclusion**: Recap + next steps + final CTA
- [ ] **Related Articles**: 3 article cards at the end

## âœ… SEO Checklist

### Meta Tags
- [ ] Title: 50-60 characters, keyword at start
- [ ] Description: 150-160 characters, compelling + keyword
- [ ] Keywords meta tag: 8-10 relevant keywords
- [ ] Robots meta: index, follow, max-snippet
- [ ] Author meta tag
- [ ] Canonical URL set correctly
- [ ] Hreflang tags (en, x-default)

### Open Graph
- [ ] og:type = "article"
- [ ] og:title (can differ from page title)
- [ ] og:description
- [ ] og:url (full URL)
- [ ] og:image (logo or featured image)

### Twitter Card
- [ ] twitter:card = "summary_large_image"
- [ ] twitter:title
- [ ] twitter:description  
- [ ] twitter:image

### Schema Markup
- [ ] BlogPosting type
- [ ] Headline
- [ ] Description
- [ ] Author (Organization)
- [ ] Publisher with logo
- [ ] datePublished
- [ ] dateModified
- [ ] mainEntityOfPage
- [ ] articleSection
- [ ] keywords array
- [ ] wordCount
- [ ] timeRequired

### On-Page SEO
- [ ] Target keyword in H1
- [ ] Target keyword in first 100 words
- [ ] Target keyword in 3+ H2s
- [ ] Related keywords distributed naturally
- [ ] 8-12 internal links to service pages
- [ ] 3-5 external links to authority sites
- [ ] All images have descriptive alt text
- [ ] Proper heading hierarchy (H1â†’H2â†’H3, no skips)
- [ ] URL is keyword-rich and clean

## âœ… Content Quality Checklist

- [ ] Answers user intent completely
- [ ] Provides unique insights (not just rehash)
- [ ] Includes real examples and case studies
- [ ] Uses current statistics (< 2 years old)
- [ ] Facts verified and accurate
- [ ] No spelling or grammar errors
- [ ] Paragraphs: 2-4 sentences max
- [ ] Lists: parallel structure, consistent punctuation
- [ ] Professional but conversational tone
- [ ] Active voice preferred
- [ ] Industry jargon explained or avoided

## âœ… Visual Design Checklist

### Images
- [ ] 1 hero image after introduction (w=1200&h=400)
- [ ] 5-6 section images throughout (w=1200&h=400)
- [ ] All from Unsplash/Pexels (royalty-free)
- [ ] All have loading="lazy"
- [ ] All have descriptive alt text
- [ ] Container: rounded-lg overflow-hidden shadow-lg mb-8
- [ ] Image: w-full h-64 object-cover block

### Diagrams/Charts
- [ ] 2-3 custom visualizations (HTML/CSS/SVG)
- [ ] Process flows clear and simple
- [ ] Data visualizations accurate
- [ ] Consistent color scheme (Bumblebee theme)

### Cards & Boxes
- [ ] Info alerts: alert alert-info
- [ ] Warning alerts: alert alert-warning  
- [ ] Example boxes: card bg-base-200
- [ ] CTAs: card bg-primary text-white shadow-xl
- [ ] Consistent spacing and styling

### Typography
- [ ] H1: text-4xl md:text-5xl font-bold gradient-header
- [ ] H2: text-2xl md:text-3xl font-bold with emoji
- [ ] H3: text-xl font-semibold
- [ ] Lead paragraph: text-xl
- [ ] Body: base size (16px)

### Spacing
- [ ] Main container: py-16
- [ ] Sections: my-8 to my-12
- [ ] Cards: p-6 to p-8
- [ ] Footer spacer: py-12

## âœ… Interactive Elements Checklist

### Self-Assessment
- [ ] Checkbox quiz present
- [ ] 5-7 relevant questions
- [ ] Result/recommendation shown
- [ ] Links to relevant service

### Collapsible Sections
- [ ] Using `<details>` + `<summary>` tags
- [ ] DaisyUI classes: collapse collapse-arrow bg-base-200
- [ ] cursor-pointer on summary
- [ ] Proper padding in content (pt-4)

### Forms
- [ ] Name field (text, required)
- [ ] Email field (email, required)
- [ ] Company field (text, required)
- [ ] Dropdown/select (relevant to topic)
- [ ] Honeypot field (hidden website field)
- [ ] Math verification (simple addition)
- [ ] Submit button: btn btn-primary btn-lg w-full
- [ ] Clear value proposition above form
- [ ] Action="/api/contact" method="POST"

### Navigation
- [ ] Table of contents with anchor links
- [ ] All section IDs match TOC hrefs
- [ ] Smooth scroll behavior
- [ ] Links work on click

## âœ… Conversion Checklist

### CTA Placement (3 locations)
- [ ] **Mid-article** (~40% through): Soft offer
- [ ] **Assessment form** (~70% through): Lead capture
- [ ] **Final CTA** (~90% through): Strong offer

### CTA Quality
- [ ] Prominent visual design (bg-primary, shadow)
- [ ] Clear headline (benefit-focused)
- [ ] Compelling copy (value proposition)
- [ ] Multiple buttons (primary + secondary)
- [ ] Relevant to surrounding content
- [ ] Not pushy/salesy

### Service Links
- [ ] Fractional BizDev Team linked
- [ ] Market Foundation Program linked
- [ ] Market Accelerator Program linked
- [ ] Contact page linked multiple times
- [ ] Solutions page linked
- [ ] Case studies/success stories linked

## âœ… Pre-Launch Checklist

- [ ] Read full article out loud (catch awkward phrasing)
- [ ] Test all interactive elements
- [ ] Test on mobile device
- [ ] Test in different browsers (Chrome, Safari, Firefox)
- [ ] Validate schema markup (Google Rich Results Test)
- [ ] Check internal links go to correct pages
- [ ] Verify external links open in new tab (where appropriate)
- [ ] Run through Grammarly or similar tool
